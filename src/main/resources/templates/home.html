Explicação + formulário teste

GET: O método GET é usado para recuperar informações de um recurso específico no servidor. É considerado um método seguro e idempotente, o que significa que pode ser executado várias vezes sem alterar o estado do recurso. Os dados são enviados através da URL como parâmetros de consulta (query parameters). Normalmente, é usado para solicitar e exibir informações.

POST: O método POST é usado para enviar dados ao servidor para criar um novo recurso. Ao contrário do método GET, os dados são enviados no corpo da requisição e não são exibidos na URL. O POST é usado para enviar informações sensíveis ou grandes quantidades de dados que não devem ser expostos na URL. Ao enviar a mesma requisição POST várias vezes, diferentes recursos podem ser criados no servidor.

DELETE: O método DELETE é usado para solicitar a exclusão de um recurso específico no servidor. Ele indica que o cliente deseja remover o recurso associado à URL fornecida. O DELETE é um método idempotente, o que significa que fazer várias solicitações DELETE para o mesmo recurso não terá efeito além da primeira exclusão.

PUT: O método PUT é usado para enviar dados ao servidor para atualizar um recurso específico. Ele substitui integralmente o recurso existente pelo novo recurso enviado no corpo da requisição. O PUT também é considerado idempotente, ou seja, fazer várias solicitações PUT para o mesmo recurso não terá efeito além da primeira atualização.
